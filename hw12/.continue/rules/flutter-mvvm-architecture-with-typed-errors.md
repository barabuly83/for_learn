---
globs: "**/*.dart"
description: Правила для реализации архитектуры MVVM с типизированными ошибками
  и правильным разделением ответственности.
alwaysApply: false
---

1. Использовать типизированные ошибки (enum или sealed classes) вместо строк в бизнес-логике
2. ViewModels не должны иметь const конструкторы если наследуются от ChangeNotifier
3. Services могут иметь const конструкторы только если все зависимости константны
4. Бизнес-логика должна возвращать типизированные результаты, а UI слой преобразует их в текст
5. Использовать freezed для создания sealed classes для ошибок и моделей данных
